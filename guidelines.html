Bitcoin Payment Guidelines

These guidelines will help you set up a Bitcoin payment page that is secure, user-friendly, and functional.

1. UI Structure and Clarity

  • Clear Instructions: Clearly state what the user needs to do on the page (e.g., "Send the specified Bitcoin amount to this address to decrypt your files").
  • Visibility of Bitcoin Address: Display the Bitcoin address prominently. Avoid making users scroll or click additional buttons to view it.
  • QR Code Option: The QR code is convenient for users paying via mobile wallets. Ensure the QR code is generated accurately and includes the Bitcoin address and correct amount if possible.

2. Bitcoin Address and QR Code

  • Use Static QR Code: If you’re displaying a static QR code, generate it with a trusted QR code generator.
  • Correct Address Format: Ensure the Bitcoin address format is correct. Bitcoin addresses are typically between 26-35 alphanumeric characters and should always start with "1," "3," or "bc1."
  • QR Code Size and Readability: Ensure the QR code is large enough and has sufficient contrast for easy scanning.

3. Payment Amount and Deadline

  • Display Payment Amount: Clearly indicate the required Bitcoin amount. Avoid changing it dynamically to prevent confusion.
  • Countdown Timer: For countdowns, provide a clear indication of the remaining time. Use a format that’s easy to read (e.g., days, hours, minutes, and seconds).

4. Encryption and Decryption Keys

  • RSA Key Generation: Use cryptographic functions provided by modern browsers (e.g., window.crypto.subtle) for secure RSA key generation.
  • Store Keys Securely: Do not store private keys or sensitive information in plain text. If using localStorage or sessionStorage, ensure sensitive data is removed after use.
  • Display Decryption Key Securely: Show the decryption key only when necessary and avoid keeping it visible longer than required.

5. Modal for Decryption Key

  • Use Modal Pop-Ups for Privacy: To protect user privacy, show the decryption key in a modal window and give users the option to close it when done.
  • Action Confirmation: Confirm with the user before redirecting or closing the page to avoid accidental data loss.
  • Redirect to Safe Pages: After showing the decryption key, redirect to a safe page (like a welcome or summary page).

6. Security Practices

  • Avoid Hardcoding Keys: Do not hardcode encryption or decryption keys in the JavaScript code. Generate them dynamically if needed.
  • HTTPS Only: Always serve the page over HTTPS to prevent man-in-the-middle attacks.
  • Do Not Store Sensitive Information in Plain Text: Use secure storage mechanisms and encrypt sensitive data if stored.

7. Error Handling

  • Expiration Alerts: Provide a clear alert if the payment deadline has passed and guide users on the next steps.
  • Modal Close Events: Ensure that the modal can be closed by clicking outside or by pressing an “X” button, so it’s easy to close.
  • Display Meaningful Error Messages: Show error messages if decryption fails or if there’s an issue with payment completion.

8. JavaScript and UI Interactivity

  • Asynchronous RSA Key Handling: When generating or decrypting RSA keys, handle these actions asynchronously. Use async and await to avoid blocking the main thread.
  • User Feedback on Payment Completion: When the "Pay" button is clicked, show a loading indicator or feedback message if the decryption takes time.
  • Optimize Performance: Avoid redundant calculations (such as repeatedly querying the countdown timer) to optimize the page’s performance.

9. User Guidance and Help

  • Provide Help Information: Include brief instructions or a “Help” link for users unfamiliar with Bitcoin payments.
  • Instructions for QR Code: Explain how to use the QR code if some users may not be familiar with it.
  • Back to Main or Home Button: Offer a clear way for users to navigate back to the main site or home page after payment is completed.
